其他功能

除了计算功能外，服务器可为用户提供其他方便的服务。如果有需求请联系管理员获取 账号。

GitLab

GitLab 是一个开源项目，使用它可以在自己的服务器上搭 建私人的 Git 平台。在功能上它和 GitHub 相似，用户可为每个 repo 独立设置访问 权限。

平台地址：https://admin-bicmr.cn/gitlab

开通账号

  • 联系管理员，提供以 pku.edu.cn 结尾的邮箱，并自行设置用户名。

  • 管理员确认后，会向申请所用的邮箱发送激活邮件，如果没收到检查一下垃圾邮件。

  • 点击邮箱中的链接，设置初始密码，完成激活。

注：非组内人员谢绝打扰，这是私有平台。想要账号的去 GitLab 官网自己注册一个，不要 骚扰管理员。如果注册失败需要检查你是否能够正常访问 Google。 GitLab 官网地址

访问范围

  • GitLab 网页端只能在北大校内访问，在校外需要通过 VPN 连接。

  • 如果仅使用 SSH 进行 repo 的各种操作，可在免费地址范围内访问。

使用注意事项

  • Windows 用户建议使用 SourceTree 客户端 进行连接。

  • Linux 用户直接使用命令行的 git 即可。如果提示找不到命令可以使用 apt 或 yum 进行安装。

  • 不单独编写教程，推荐新用户看下这个文档熟悉一下 git，至于在 GitLab 网页端上的操作则十分简单。

  • 如果忘记密码，在登录界面点击 Forgot your password，而后输入注册时使用的 邮箱即可收到重置密码的邮件。

Seafile

Seafile 是一个开源的私人文件云平台，功能上与 Dropbox 相似。这个平台可以基本上 解决组内成员共享文件的问题。

平台地址：https://file.admin-bicmr.cn

主要功能

  • 文件存储。功能类似于个人云盘，用户可以将本地文件存储到组内 Seafile 上备用。

  • 文件同步。使用客户端后，资料库将会在本地和云端自动同步，保留较新版本。 用户可设置要同步哪些资料库。

  • 文件共享。用户可以和平台其他用户共享资料库或文件夹，被共享者拥有只读权限 或读写权限。

  • 下载/上传链接。用户可生成下载或上传链接。两种链接均可被任何人访问（无论 他们是否是平台用户）。使用上传链接时上传者不能查看别人上传的文件。

  • 历史版本。所有文件的历史版本都会被平台记录，方便回滚。

开通账号

  • 联系管理员，提供以 pku.edu.cn 结尾的邮箱。

  • 管理员确认后，会为你设置随机的初始密码。

  • 打开网页端，使用邮箱+密码登录，修改密码。

访问范围

  • Seafile 平台（包括生成的链接）只能通过校内访问，校外需要使用 VPN。

使用注意事项

  • Seafile 需要配合客户端使用。一般只需要下载一个同步客户端。下载地址

  • 在配置客户端的时候，服务器地址填写 https://file.admin-bicmr.cn，不要漏了 前面的 https。

  • 目前暂无配额，慎用下载链接和上传链接。无特殊情况不得向组外人员以链接方式 提供资源和服务。

  • 如果忘记密码直接让管理员进行重置。Web 端的重置密码功能没开。
